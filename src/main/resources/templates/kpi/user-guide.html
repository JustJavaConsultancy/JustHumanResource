KPI Appraisal Reports

KPI & Appraisal User Guide

Complete operational guide for KPI definition, assignment, measurement, appraisal workflow, and KPI/appraisal reporting.

Module Overview

The KPI module runs as a sequence: Definition → Assignment → Measurement → Appraisal → Reports. The appraisal workflow includes manager review and employee self-review, then final outcome calculation.

  • Primary screen: /kpi (tabs: Definitions, Assignments, Measurements, Appraisal)
  • Reports screen: /kpi-report/dashboard and related report pages
  • API workflow endpoints: /api/appraisals/*

1) KPI Definition

  1. 1 Open KPI Definitions tab.
  2. 2 Create KPI with: Code, Name, Description, Category, Target Value, Unit, and Impact Salary.
  3. 3 Submit. Newly created KPI is automatically marked active.

Categories include Productivty/Attendance/Quality and units include Percentage/Number/Hours.

2) KPI Assignment

  1. 1 Open KPI Assignments tab and choose scope: Employee, Job Step, or Department.
  2. 2 Add one or more KPI rows with weight and mandatory flag.
  3. 3 Submit via /kpi/assign.
Weight rule: total active weight for the selected scope cannot exceed app.kpi.max-kpi-weight (default 1.0, environment override MAX_KPI_WEIGHT).

3) KPI Measurement

  1. 1 Open KPI Measurements.
  2. 2 Select employee to load only effective assigned KPIs.
  3. 3 Enter period (YYYY-MM) and actual values; submit.
  4. 4 Existing same employee/KPI/period entries are skipped; values can be edited from the measurements table.

Actual value must be zero or positive.

Measurement Scoring Logic

  • Per KPI score: (actual / target) * 100, capped at 100.
  • If target ≤ 0: score is 0.
  • Salary-impact KPI score: average of KPIs where impactSalary=true; if none, fallback score = 100.

4) Appraisal Flow (Manager + Employee)

A. Appraisal Cycle

Create cycles via API POST /api/appraisals/cycles with year and quarter (Q1–Q4). A cycle spans 3 months.

B. Draft Appraisal

Draft appraisal computes KPI score from measurements in cycle period. Endpoint: POST /api/appraisals/draft.

C. Manager Review

Manager enters score/comment from the Appraisal tab panel and submits through workflow task completion.

D. Employee Self-Review

Employee self-score/comment is captured in workflow and displayed in appraisal records for manager visibility.

Final Appraisal Calculation

  • KPI score: average of measurement scores in cycle range.
  • Final score formula: (kpiScore * 0.7) + (managerScore * 0.3).
  • If manager score is missing, final score falls back to KPI score.
  • Finalization endpoint: POST /api/appraisals/finalize.

Outcome Bands

  • EXCEEDS_EXPECTATION: score ≥ 85
  • MEETS_EXPECTATION: score ≥ 70 and < 85
  • NEEDS_IMPROVEMENT: score ≥ 50 and < 70
  • UNDERPERFORMING: score < 50

5) KPI & Appraisal Reports

Report Route Purpose
Dashboard /kpi-report/dashboard Overall distribution, top/bottom performers, department KPI overview.
Top Performers /kpi-report/top-performers Leaderboard by KPI and appraisal score.
Scorecard (All) /kpi-report/scorecard All employee KPI lines with weight and weighted score.
Appraisal Summary /kpi-report/appraisal Cycle appraisal outputs: KPI score, manager/self scores, final score, outcome.
Employee Scorecard Detail /kpi-report/employee-scorecard/{employeeId} Single employee detailed KPI scorecard for selected period.

Operational Checklist

  1. 1. Define KPIs with correct targets, units, and salary impact flags.
  2. 2. Assign KPIs by scope and confirm total active weight is within configured max.
  3. 3. Capture measurements each period for assigned KPIs only.
  4. 4. Create/maintain appraisal cycles and ensure measurement data exists for cycle period.
  5. 5. Complete manager and employee review tasks.
  6. 6. Review final scores, outcomes, and KPI reports for action planning.
