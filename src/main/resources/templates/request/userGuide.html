User Guide

Request Workflow

Use this guide to create, submit, approve, return, reject, track, and close organizational requests. The module supports staff requisitions, file requests, asset requests, and general requests.

1. What The Request Workflow Does

The Request Workflow is the application area for managing organization-wide requests. A requester creates a draft, submits it for approval, and tracks its status. Approvers receive tasks, make decisions, and leave comments. HR/admin users can see wider request activity and close approved requests when the work is complete.

Create

Capture request details and draft them.

Submit

Send the request into approval workflow.

Approve

Approvers approve, reject, or return requests.

Track

Review status, timeline, comments, files, and items.

2. Who Uses The Workflow

Role What They Can Do Typical Screen
Employee Create requests, save drafts, submit owned requests, track owned requests, add comments, and upload permitted attachments. /requests
Approver View assigned approval tasks and approve, reject, or return requests with comments. Pending My Approval section
HR/Admin View all requests, add internal comments, and close approved requests. /requests and request detail pages

Note: every user who creates a request must have an employee profile linked to their login email.

4. Request Types

Type Use It For Key Fields Special Rule
Staff Requisition Request new or replacement headcount. Job title, department ID, positions, employment type, target start date, budget, reason for hire. Requires staff requisition details before it can be created.
File Request Request access to, approval for, or processing of files/documents. File category, confidentiality level, requested access type, purpose, retention flag. Configured to require at least one attachment before submission.
Asset Request Request equipment, software, tools, supplies, or other assets. Cost center, required date, business justification, item rows. Requires at least one item.
General Request Capture requests that do not fit another category. Title, description, priority. Minimal required fields.

5. Create And Submit A Request

Create Draft

  1. Open Requests.
  2. Click New Request.
  3. Select the request type.
  4. Enter the common fields: title, description, and priority.
  5. Complete the type-specific fields that appear.
  6. Click Create Draft.

Submit For Approval

  1. Open the request detail page.
  2. Review the summary and type details.
  3. Add required attachments or items if needed.
  4. Click Submit.
  5. The request moves to In Approval when an approval route is found.
If the system says no approval route is configured, the requester may not have a reporting line configured.

6. Understand The Request Detail Page

Main Sections

  • Summary: title, type, requester, department, and submission date.
  • Type Details: request-type-specific data.
  • Items: asset or purchase-style item rows.
  • Attachments: files uploaded to support the request.

Tracking Sections

  • Approval Timeline: each approver level and decision.
  • Comments: requester, approver, and HR/admin notes.
  • Activity: creation, submission, approval, rejection, return, attachment, cancellation, and closure events.

7. Approval Flow

Submitted requests use the configured Flowable workflow process. The current implementation resolves approvers using the requester's line-manager approval route. Approvers see their tasks in the Pending My Approval section.

Approve

Moves the request to the next approver. If there are no more approvers, the request becomes approved.

Reject

Stops the workflow and marks the request as rejected. A rejection comment is required.

Return

Returns the request for correction. The requester can edit supporting information and resubmit.

8. Files And Items

Attachments

  • Files can be uploaded from the request detail page when the request is still editable.
  • File requests require at least one attachment before submission.
  • Supported file types include PDF, PNG, JPEG, text, Word, and Excel formats.
  • The current file size limit is 20 MB.

Items

  • Asset requests require at least one item row.
  • Use item rows for quantity, unit of measure, unit cost, currency, category, and vendor.
  • The system calculates estimated total cost from unit cost and quantity.

9. Status Guide

Status Meaning What To Do Next
Draft The request has been created but not submitted. Review details, add files/items if needed, then submit.
Submitted The request has been submitted and is preparing for workflow handling. Wait for approval processing.
In Approval The request is assigned to one or more approvers. Approver should approve, reject, or return.
Returned For Correction An approver returned the request for updates. Requester should review comments, correct details, and resubmit.
Approved All required approval steps are complete. HR/admin may close the request when fulfilled.
Rejected The request was rejected and the workflow ended. Review comments before creating a new request if needed.
Cancelled The requester cancelled the request before completion. No further action.
Closed An approved request has been completed and closed. No further action.

10. Troubleshooting

I cannot create a request

Confirm your login email is linked to an employee record. The system uses your employee record to identify request ownership.

Submission says no approval route is configured

Ask HR/admin to check your primary reporting line. Requests currently depend on the line-manager approval route.

My file request will not submit

File requests require at least one attachment before submission. Upload a supported file from the detail page, then submit again.

My asset request will not submit

Asset requests require at least one item row. Add an item with a valid name and quantity.

I cannot remove an attachment

Attachments can only be removed by the requester while the request is in draft or returned-for-correction state.

11. Recommended Working Habits

  • Use clear request titles so approvers can understand the request from the list page.
  • Attach supporting documents before submitting file-sensitive or approval-sensitive requests.
  • Use comments to explain corrections, decisions, and follow-up notes.
  • Check the approval timeline before asking for status updates.
  • HR/admin users should close approved requests after fulfillment to keep open lists clean.
